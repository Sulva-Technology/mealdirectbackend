# Meal Direct Database Access Matrix

Default access is denied. The API service uses server-only Supabase service credentials and bypasses RLS for ordinary application operations. Direct browser access is intentionally narrow and mostly read-only.

| Table or surface                                                                    | Anonymous             | Customer                          | Vendor user                          | Rider                         | Campus admin                           | Super admin     | Backend service role |
| ----------------------------------------------------------------------------------- | --------------------- | --------------------------------- | ------------------------------------ | ----------------------------- | -------------------------------------- | --------------- | -------------------- |
| profiles                                                                            | none                  | read/update own safe fields       | read own                             | read own                      | read campus profiles                   | read all        | full                 |
| campuses, campus_zones, campus_locations                                            | read active           | read active                       | read active                          | read active                   | manage own campus                      | manage all      | full                 |
| vendors, menu_categories, unit_types, menu_items, delivery_slots                    | read active catalogue | read active catalogue             | read active catalogue for own vendor | read active catalogue         | manage own campus catalogue            | manage all      | full                 |
| vendor_users                                                                        | none                  | none                              | read own vendor membership           | none                          | manage own campus                      | manage all      | full                 |
| vendor_payout_accounts                                                              | none                  | none                              | no direct read                       | none                          | no direct read                         | manage all      | full                 |
| riders                                                                              | none                  | none                              | none                                 | read own rider record         | manage own campus                      | manage all      | full                 |
| inventory and availability                                                          | none                  | read orderable availability views | read own vendor inventory            | none                          | manage own campus                      | manage all      | full                 |
| orders and order_items                                                              | none                  | read own orders                   | read own vendor orders               | read assigned delivery orders | read own campus                        | read all        | full                 |
| payments, refunds, settlements                                                      | none                  | no direct payment payload access  | no direct payout access              | no direct payout access       | read campus summaries only through API | manage all      | full                 |
| audit_logs, payment_events, status_history, settlement_lines, inventory_adjustments | none                  | none                              | none                                 | none                          | append/read scoped audit through API   | append/read all | full                 |

Security-definer functions set a fixed `search_path` and expose only stable checks or transactional operations. Privileged role assignment is never inferred from `auth.users.raw_user_meta_data`.
